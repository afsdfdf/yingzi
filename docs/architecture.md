# 架构说明

## 总体思路

项目采用 `monorepo + 模块化单体` 架构，优先保障正式版的开发效率和可维护性。

- `apps/api`：统一后端 API
- 后续可新增 `apps/admin-web`、`apps/mobile-h5`
- 按领域拆模块，而不是按端拆后端服务

## 核心领域

1. 身份与权限
2. 租户与商户
3. 商品与库存
4. 订单与售后
5. 客户与会员
6. 运营与营销
7. AI 服务
8. 数据与报表

## 正式版演进

- 第一步：Mock 数据 + API 契约
- 第二步：接入 MySQL 和 ORM
- 第三步：接入 JWT、RBAC、操作审计
- 第四步：接入管理端和用户端
- 第五步：接入支付、短信、AI 能力
