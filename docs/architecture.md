# 架构说明

## 总体思路

项目采用 `monorepo + 模块化单体` 架构，优先保障正式版开发效率、可维护性和后续扩展能力。

- `apps/api`：统一后端 API 服务
- `apps/admin-web`：平台总后台与商户后台前端
- `apps/mobile-h5`：面向用户端的移动 H5
- `docs/`：架构、部署、数据库等项目文档

当前阶段优先按“业务域”拆分后端能力，而不是按端拆多个后端服务。这样可以最大化复用商户、商品、订单、活动、用户等核心能力。

## 核心领域划分

1. 身份与权限
2. 租户与商户
3. 店铺与员工
4. 商品与库存
5. 订单与售后
6. 客户与会员
7. 运营与营销
8. AI 服务与数据分析

## 端与职责

### API

统一承载以下能力：

- 登录认证与 JWT
- RBAC 角色权限控制
- 平台管理接口
- 商户管理接口
- 用户端商城接口
- 后续数据库、缓存、文件上传、支付、AI 接入

### Admin Web

主要面向：

- 平台超级管理员
- 平台运营管理员
- 商户老板
- 店长与店员

核心目标：

- 高效后台操作
- 统一数据总览
- 商品、订单、商户、活动、用户等管理

### Mobile H5

主要面向：

- 普通消费者
- 会员客户

核心目标：

- 逛店、逛商品、下单、查看订单
- 后续接入登录、地址、优惠券、个人中心等能力

## 正式版演进路径

### 第一阶段：平台底座

- Mock 数据与 API 契约
- 登录、JWT、RBAC
- 平台总览、商户、商品、订单、用户基础接口

### 第二阶段：前端管理台与用户端

- Admin Web 管理后台
- Mobile H5 商城页面
- 打通基础交互链路

### 第三阶段：数据持久化

- 接入 MySQL
- 引入 ORM
- 将商品、订单、商户、活动等改为真实数据库读写

### 第四阶段：业务闭环

- 购物车、下单、订单中心
- 商户管理、商品管理、活动管理
- 图片上传、状态流转、表单校验

### 第五阶段：生产化能力

- BaoTa / Linux 部署
- PM2 与 Nginx
- 环境变量管理
- 日志、监控、备份

### 第六阶段：增强能力

- AI 客服与 AI 辅助生成
- 营销玩法扩展
- 多业态业务插件化扩展
